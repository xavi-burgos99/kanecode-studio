Inputs

KaneCode Studio uses a class (KCStudioInput) to generate the differents inputs.

KCStudioInput Class

This class contains differents methods to configure any type of input with only a single piece of code (using a JSON object).

Using JavaScript
new KCStudioInput(studio, 1, 'My first input', 'text', 'general', {
	placeholder: 'Example',
	value: (studio) => studio.selected.id,
	onChange: (studio, value) => { studio.selected.id = value }
});

Using JSON
{
	label: {
		"en-US": "My first input",
		"es-ES": "Mi primer input"
	},
	placeholder: {
		"en-US": "Example",
		"es-ES": "Ejemplo",
	},
	type: "text",
	order: 10,
	section: "general",
	value: (studio) => studio.selected.id,
	onChange: (studio, value) => { studio.selected.id = value }
}

Options

Option Types Default Description
placeholder string, number, function undefined Placeholder of the input.
value string, number, object, function undefined Value of the input.
onChange string, number, object, function undefined Function to call when the value of the input changes. It accepts also the value of the input.

Methods

Method Modifiable Description
constructor(studio, id, label, type, section, options) No Constructor method.
addTrigger(element, eventType) No Adds a event trigger to an input.
checkValueFormat(value) Yes Checks if the value is correct. It accepts any type of value.
generator(options) Yes Initializes the HTML input. Includes all the necessary elements and inputs to configure it and add the different triggers.
elementToValue() Yes Stores the value obtained from the option value in a class variable.
inputToValue() Yes Stores the value of the input in a class variable.
refresh(value) No Refreshes the input. It accepts any type of value. If the checked value is not correct, the method will receive the value directly from the element (using the method elementToValue()).
trigger() No Triggers the option onChange if it is correct.
valueToInput() Yes Specifies the value of the input. It accepts any type of value, even if it is a function.
